## 一、 效能指标（Performance）——“跑得快不快”

| 指标 | 定义 | 目标 | 当前状态 (V37.0) |
| :--- | :--- | :--- | :--- |
| **端到端时延** | 用户提问到看到完整报告 | < 20s | **12.8s (P90) ✅ 已达标** |
| **首字延迟 (TTFT)** | 用户看到第一个字符的时间 | < 2s | **1.8s ✅ 已达标** |
| **节点耗时分布** | Planner/Coder/Auditor/Reporter 占比 | 均衡分布 | 核心瓶颈已移至 LLM 推理 |
| **Token 生成速率** | 模型生成 token 的速度(tokens/s) | > 50 | 约 42.5 tokens/s (受限于 API) |
| **冷启动开销** | 模型/索引从加载到就绪的时间 | < 5s | **0.01s (常驻单例) ✅ 已达标** |
| **吞吐量** | 单位时间处理的任务数 | 待定 | 支持高并发长连接 |

**当前突破点**：通过 **Singleton (单例常驻)** 与 **Hyper-Fast Bypass** 彻底解决了 20s 冷启动问题。

*   **现状**: 链路完整，Auditor 决策过程已进入 log 流水。

*   **目标**: > 70%。
*   **现状**: 依靠 `app/experience` 模块进行锦囊匹配。
